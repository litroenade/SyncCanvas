# SyncCanvas - AI驱动的实时协作白板系统

## 一、项目详情及使用方式

### 1.1 项目概述

**SyncCanvas** 是一个基于 CRDT（Conflict-free Replicated Data Type）技术的**实时协作白板应用**，支持多人同时在线编辑。与传统白板工具不同的是，SyncCanvas 集成了**AI Agent**，允许用户通过**自然语言**直接控制画布元素，而不是手动绘制或生成静态图表。

**核心理念**：
- **直接控制画布**：AI不生成Mermaid代码或元组，而是直接操作已存在的画布元素
- **自然语言交互**：用户通过对话即可创建、修改和布局图表
- **实时协作**：多人同时编辑，数据实时同步，无冲突
- **类似DrawIO**：采用类似DrawIO的元素存储方式（mxCell），但增加了多人协作能力

### 1.2 技术架构

## 四、项目中涉及的关键技术

### 4.1 CRDT（Conflict-free Replicated Data Type）

#### 技术原理
CRDT是一种**无需中央协调即可保证最终一致性**的数据结构。

**实现方案**：
- **前端**：Yjs（JavaScript CRDT库）
- **后端**：pycrdt（Python CRDT库）
- **通信**：y-websocket + pycrdt-websocket

**工作流程**：
```
客户端A修改 → Yjs生成操作 → WebSocket传输 → 后端Y.Doc应用
                                    ↓
客户端B接收 ← WebSocket广播 ← 所有其他客户端
```

**代码示例**：
```python
# 后端 - WebSocket同步服务
class PersistentWebsocketServer(WebsocketServer):
    """带持久化的CRDT WebSocket服务器"""
    
    async def get_room(self, name: str) -> YRoom:
        """获取或创建房间"""
        if name not in self.rooms:
            # 创建Y.Doc + SQLite持久化
            ystore = SQLModelYStore(name, engine, Doc)
            self._ystores[name] = ystore
            
            # 初始化画布后端
            canvas_backend = init_canvas_backend(name)
            room = YRoom(ystore=ystore, ymetadata=canvas_backend)
            
            self.rooms[name] = room
        return self.rooms[name]
```

**优势**：
- ⚡ 无需锁，性能高
- 🔄 自动冲突解决
- 💾 断线重连数据恢复
- 📡 低延迟同步

### 4.2 ReAct (Reasoning + Acting) AI框架

#### 框架设计

**状态机**：
```
INITIALIZING → THINKING → ACTING → OBSERVING → THINKING → COMPLETED
                  ↑           ↓
                  └─── ERROR ←┘
```

**执行循环**：
```python
class BaseAgent:
    async def run(self, prompt: str) -> dict:
        """ReAct主循环"""
        # 1. 初始化
        self._state_machine.transition_to(AgentState.THINKING)
        
        # 2. 迭代执行
        while not self._should_stop():
            # THINKING阶段：调用LLM
            response = await self.llm.chat(
                messages=self._build_messages(),
                tools=self._get_tool_schemas()
            )
            
            # ACTING阶段：执行工具
            if response.tool_calls:
                self._state_machine.transition_to(AgentState.ACTING)
                for tool_call in response.tool_calls:
                    result = await self._execute_tool(tool_call)
                
                # OBSERVING阶段：收集结果
                self._state_machine.transition_to(AgentState.OBSERVING)
            
            # 判断是否完成
            if response.finish_reason == "stop":
                self._state_machine.transition_to(AgentState.COMPLETED)
                break
```

### 4.3 图布局算法

#### 拓扑排序 + 层级布局

**算法流程**：
```python
def calculate_layout(structure: dict, theme: str) -> LayoutResult:
    """主布局引擎"""
    # 1. 构建邻接表
    adj = build_adjacency(nodes, edges)
    
    # 2. 拓扑排序（Kahn算法）
    levels = topological_levels(nodes, adj)
    
    # 3. 分配层级和列
    for level, level_nodes in enumerate(levels):
        for col, node_id in enumerate(level_nodes):
            node.level = level
            node.column = col
    
    # 4. 计算坐标
    for node in layout_nodes:
        node.x = start_x + node.column * (node_width + h_gap)
        node.y = start_y + node.level * (node_height + v_gap)
    
    # 5. 生成连接箭头
    edges = generate_arrows(layout_nodes, adj)
    
    return LayoutResult(nodes=nodes, edges=edges)
```

2. **CRDT实时协作** - 工业级的多人同步方案
3. **ReAct框架** - 可靠的AI决策系统
4. **图布局算法** - 自动化的视觉优化
5. **全栈TypeScript/Python** - 类型安全的开发体验
